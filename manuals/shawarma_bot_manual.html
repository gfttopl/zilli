1. 🤖 About the bot

Shawarma Order Bot is a Telegram bot that lets users order shawarma with a simple menu + cart, then sends the final order to an admin group.

  • Pickup only (no delivery).
  • Asks for phone number once and stores it.
  • Has categories: shawarma, sauces, extras, drinks.
  • Supports weekly promos (example: 2nd sauce free).

2. ✅ What the bot can do

Feature How it works
Phone capture & storage On first /start the user shares contact. Phone is saved in phones.txt and reused later (no re-share).
Main menu Buttons: Order, Promotions, Order History, Info.
Fast ordering screen Shows categories and a mini cart preview. One button moves to cart.
Cart management In cart: increase/decrease/remove items; buttons: Add more, Place order.
Promotions logic Example: sauces cost 25 грн, but the 2nd sauce in the same order becomes 0 грн automatically.
Admin notification On order, bot sends message to admin group with time, items, total, and clickable phone (tel: link).
Order history Saves each order to orders_log.jsonl. Users can view last 5 orders.
Info screen Shows address, working hours, shop phone, pickup message.
Tech commands /chatid prints the current chat id (useful for admin group setup).

3. 🧾 User flow (how to order)

  1. User opens bot and sends /start.
  2. If phone is not saved yet, user taps “Share phone”.
  3. User taps Order → sees categories and mini cart.
  4. User adds items in categories; mini cart updates.
  5. User taps Go to cart, checks items, adjusts quantities.
  6. User taps Place order (pickup).
  7. Bot confirms success and shows shop phone for changes.
💡 Tip: For the smoothest flow, pin the bot link/QR in your shop and keep the admin group ID configured.

4. 🛠 Admin setup

4.1 Create admin group and get chat_id

  1. Create a Telegram group for admins.
  2. Add the bot to that group.
  3. Send /chatid in the group to get the ID.
  4. Put it into ADMIN_CHAT_ID in .env.

4.2 .env example

BOT_TOKEN=123456:ABCDEF...
ADMIN_CHAT_ID=-1001234567890

4.3 Run on Mac (test)

python3 -m venv .venv
source .venv/bin/activate
pip install python-telegram-bot python-dotenv
python3 bot.py

5. 📁 Files & data storage

  • bot.py — main code.
  • .env — bot token and admin group id.
  • phones.txt — phone database (one line per user).
  • orders_log.jsonl — order history in JSON lines format.

6. ✏️ Editing menu & prices

Edit these lists in bot.py:

SHAWARMA_ITEMS = [("Класична (міні)", 120), ...]
SAUCES = [("Часниковий", 25), ...]
EXTRAS = [("Дод. сир", 20), ...]
DRINKS = [("Coca-Cola 0.5", 35), ...]
⚠️ Note: Inline keyboard buttons do not support HTML. That’s why “25 → 0” in sauce promo uses a unicode strike-through.

7. ❓ FAQ

Why does the bot ask for the phone?

So the admin can contact the customer if something needs to be уточнено/changed. Phone is stored once and reused.

Can I add delivery later?

Yes. Add a new step to ask for address and include it into admin message + order log.

Where is weekly promo shown?

In “Promotions” screen and inside cart when promo is applied.

8. 🧯 Troubleshooting

ModuleNotFoundError: dotenv

Install python-dotenv: python3 -m pip install python-dotenv.

Orders are not sent to admin group

Check ADMIN_CHAT_ID and make sure the bot is in the group.

Message is not modified error

The code uses safe edit to ignore this Telegram warning.
